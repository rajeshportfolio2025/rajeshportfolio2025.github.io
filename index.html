Git Concepts

Fundamental version control concepts and commands I've learned in my studies.

Version Control

The practice of tracking and managing changes to code over time, allowing multiple developers to collaborate efficiently.

git init
git clone [repository]

Branching

Creating isolated environments to develop features without affecting the main codebase.

git branch [branch-name]
git checkout [branch-name]
git merge [branch-name]

Stashing Changes

Temporarily shelving changes you've made to your working directory so you can work on something else.

git stash
git stash pop
git stash list

.gitignore

A file that specifies intentionally untracked files that Git should ignore (like node_modules, .env files, etc.).

# .gitignore example
node_modules/
.env
*.log

Rebasing

Rewriting commit history by moving or combining commits, creating a linear project history.

git rebase [base-branch]
git rebase -i HEAD~3

Cherry-Picking

Applying the changes introduced by some existing commits to the current branch.

git cherry-pick [commit-hash]

Squashing Commits

Combining multiple commits into one for cleaner history, often done during rebase.

git rebase -i HEAD~5
# Then mark commits with "squash" or "fixup"

Reflog

A safety net that records when the tips of branches and other references were updated.

git reflog
git reset --hard [reflog-hash]
